## 1. 한 줄 요약

PyQt5 기반 데스크탑 펫 **데모**. 사용자가 목표를 입력하면 사자 캐릭터가 우하단에 떠다니며, 5초마다 활성 창 제목을 확인해 목표 키워드가 보이면 칭찬, 안 보이면 서운한 말풍선을 띄움. "완료" 버튼 클릭 시 축하 메시지.

**용도**: 영상 1편짜리 시연. 정확성/확장성/AI 호출 일체 제외.

---

## 2. 사용자 흐름

1. 앱 실행
2. **"오늘 목표?"** 한 줄 입력 다이얼로그 (예: `Python 공부`)
3. 입력 받은 후 다이얼로그 닫힘 → 우하단에 사자 등장
4. 사자 옆 또는 아래에 작은 **"완료" 버튼** 표시
5. **5초마다** 활성 창 제목을 읽어서:
   - 제목에 목표 키워드가 substring으로 포함 → **칭찬 메시지** 중 랜덤 (말풍선 3초)
   - 미포함 → **서운 메시지** 중 랜덤 (말풍선 3초)
6. 사용자가 "완료" 클릭 → **축하 메시지** 출력 후 5초 뒤 앱 종료

## 11. 알려진 함정 (이 정보 모르면 시간 낭비)

### 11.1 Ctrl+C 가 PyQt5 앱에서 안 먹힘 (Windows)

PyQt5의 Qt 이벤트 루프가 C++에 잠겨 있어 Python signal handler가 못 깨어남. 사용자가 터미널에서 Ctrl+C 눌러도 앱이 안 죽음.

**해결**: `QApplication` 생성 직후 다음 코드 추가:

```python
import signal
from PyQt5.QtCore import QTimer

signal.signal(signal.SIGINT, signal.SIG_DFL)   # OS 기본 동작(즉시 종료)으로 등록
sigint_timer = QTimer()                        # 빈 타이머 — Python에 제어권 양보용
sigint_timer.start(500)                        # 500ms 마다
sigint_timer.timeout.connect(lambda: None)     # 아무 일도 안 함 (목적: 신호 검사 기회 제공)
```

### 11.2 Microsoft Store Python 더미 주의

`where python` 결과 첫 번째가 `C:\Users\user\AppData\Local\Microsoft\WindowsApps\python.exe` 라면 그건 **MS Store 더미** (실행하면 그냥 종료). 진짜 Python은 위 경로(`Python312\python.exe`)를 직접 사용하거나 `.venv` 활성화 후 `python` 명령 사용.

### 11.3 `WA_TranslucentBackground` 필수

데스크탑 펫이 흰 사각 배경 없이 진짜 떠다니려면 `Qt.WA_TranslucentBackground` 속성 필수. 이미지/이모지 위젯 모두 적용.

### 11.4 활성 창 자기 자신 잡기

`watcher.py` 가 활성 창 제목을 검사할 때, 우리 앱 자체 창이 활성이면 자기 자신을 검사하게 됨. 사자 위젯의 창 제목을 비워두거나, 검사 시 자기 PID 제외하는 처리 권장.
